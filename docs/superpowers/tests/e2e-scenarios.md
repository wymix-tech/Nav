# 个人导航页 — E2E 测试场景

## 场景 1：首次使用（无后端）

**用户故事：** 用户首次打开导航页，在无后端的情况下配置个人导航页。

### 步骤

1. 打开 http://localhost:3000
2. 验证页面正常加载，显示空画布
3. 验证顶栏显示"编辑"按钮（无登录按钮）
4. 点击"编辑"进入编辑模式
5. 验证右侧出现组件库面板
6. 点击"搜索框"添加到画布
7. 点击"时钟"添加到画布
8. 验证两个组件正确渲染
9. 拖拽搜索框调整位置
10. 拖拽时钟边缘调整大小
11. 点击"完成编辑"退出编辑模式
12. 验证组件正常展示，编辑 UI 隐藏
13. 刷新页面
14. 验证布局和组件保持不变（IndexedDB 持久化）

**预期结果：** 用户完成个人导航页配置，数据本地持久化。

---

## 场景 2：后端登录与同步

**用户故事：** 用户启动后端服务，登录后数据同步到服务器。

### 步骤

1. 启动后端：`NAV_JWT_SECRET=secret NAV_PASSWORD=admin123 pnpm dev:server`
2. 启动前端：`pnpm dev:frontend`
3. 打开 http://localhost:3000
4. 验证顶栏显示"登录"按钮
5. 点击"登录"，输入密码 `admin123`
6. 验证登录成功，顶栏显示"编辑"和"退出登录"
7. 点击"编辑"进入编辑模式
8. 添加搜索框和时钟组件
9. 配置搜索框引擎为 Baidu
10. 点击"完成编辑"
11. 验证数据已保存到后端（SQLite）
12. 打开另一个浏览器（或无痕模式）
13. 访问同一地址，登录
14. 验证显示相同的布局和组件配置

**预期结果：** 数据跨浏览器同步。

---

## 场景 3：第三方组件安装

**用户故事：** 用户从 GitHub 安装第三方组件。

### 步骤

1. 登录并进入编辑模式
2. 点击组件库中的"安装组件"
3. 输入一个有效的 GitHub 仓库地址（如包含 manifest.json 的测试仓库）
4. 点击"预览"
5. 验证显示组件信息（名称、版本、作者、描述）
6. 点击"安装"
7. 验证组件出现在已安装组件列表
8. 点击已安装组件添加到画布
9. 验证组件从 CDN 加载并渲染
10. 退出编辑模式
11. 验证第三方组件正常展示

**预期结果：** 第三方组件从 GitHub CDN 动态加载并正常工作。

---

## 场景 4：响应式布局

**用户故事：** 用户在不同设备上查看导航页。

### 步骤

1. 在桌面端（1920x1080）配置布局
2. 添加 4 个组件，调整位置和大小
3. 记录 lg 布局状态
4. 缩小浏览器窗口至 768px 宽度
5. 验证自动切换到 sm 断点（6 列）
6. 验证组件按 sm 布局排列
7. 继续缩小至 375px 宽度
8. 验证自动切换到 xs 断点（4 列）
9. 验证组件按 xs 布局排列，无重叠
10. 在 xs 断点下调整组件位置
11. 恢复桌面宽度
12. 验证 lg 布局保持不变

**预期结果：** 各断点布局独立，互不干扰。

---

## 场景 5：权限控制

**用户故事：** 未登录用户只能查看，不能编辑。

### 步骤

1. 启动后端，打开页面（不登录）
2. 验证顶栏显示"登录"按钮
3. 验证组件只读展示，无编辑 UI
4. 尝试直接调用 API（curl PUT /api/dashboards/default）
5. 验证返回 401 未授权
6. 登录后重复步骤 4
7. 验证 API 调用成功

**预期结果：** 未登录用户只能读取，所有写操作需要认证。

---

## 场景 6：组件配置

**用户故事：** 用户配置天气组件的 API Key 和城市。

### 步骤

1. 登录并进入编辑模式
2. 添加天气组件到画布
3. 点击天气组件的配置按钮（⚙）
4. 验证弹出配置表单，显示 API Key、城市、单位字段
5. 输入 OpenWeatherMap API Key
6. 输入城市为"上海"
7. 关闭配置弹窗
8. 验证天气组件显示上海天气
9. 修改城市为"东京"
10. 验证天气组件更新为东京天气

**预期结果：** 配置表单自动生成，配置变更实时生效。

---

## 场景 7：错误处理

**用户故事：** 用户在各种异常情况下的体验。

### 步骤

1. **无效仓库地址：** 安装组件时输入 `https://github.com/invalid/repo`
   - 预期：显示"无法获取 manifest.json"

2. **manifest 格式错误：** 安装缺少必填字段的仓库
   - 预期：显示"manifest.json 格式不正确"

3. **组件加载失败：** 安装 entry 文件不存在的组件
   - 预期：显示加载错误，不影响其他组件

4. **网络断开：** 断网后操作
   - 预期：本地操作正常，同步操作提示失败

5. **Token 过期：** 登录后等待 token 过期
   - 预期：自动退出登录，提示重新登录

**预期结果：** 各种异常情况有友好的错误提示，不影响整体使用。

---

## 场景 8：完整工作流

**用户故事：** 用户从零开始完成个人导航页配置。

### 步骤

1. 清空浏览器数据
2. 打开页面（纯前端模式）
3. 进入编辑模式
4. 添加搜索框、时钟、天气、书签
5. 配置天气 API Key
6. 添加书签分组（工作、个人）
7. 调整各组件位置和大小
8. 退出编辑模式，验证展示效果
9. 启动后端
10. 登录
11. 验证数据自动同步到后端
12. 打开新浏览器
13. 登录
14. 验证所有配置同步完成
15. 安装一个第三方组件
16. 配置并使用

**预期结果：** 完整的个人导航页搭建和跨设备同步。
