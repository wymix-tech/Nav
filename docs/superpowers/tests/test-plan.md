## 测试范围

### 功能模块

| 模块 | 测试重点 | 优先级 |
|------|---------|--------|
| 布局系统 | 拖拽、调整大小、响应式断点、多端布局同步 | P0 |
| 组件系统 | 内置组件渲染、第三方组件安装/加载、配置表单 | P0 |
| 认证系统 | 登录/登出、权限控制、无后端模式 | P0 |
| 数据层 | IndexedDB 读写、后端同步、adapter 切换 | P1 |
| 组件商店 | GitHub 仓库安装、manifest 校验、更新检查 | P1 |
| UI 交互 | 编辑模式切换、组件库面板、配置弹窗 | P1 |

### 测试类型

| 类型 | 工具 | 覆盖范围 |
|------|------|---------|
| 单元测试 | Vitest | stores、services、工具函数 |
| 组件测试 | Vitest + @vue/test-utils | Vue 组件渲染和交互 |
| E2E 测试 | Playwright | 完整用户流程 |
| 手动测试 | 浏览器 | 视觉验证、响应式、跨设备 |

## 测试环境

- **前端：** http://localhost:3000（Vite dev server）
- **后端：** http://localhost:4000（Hono server）
- **浏览器：** Chrome、Safari、Firefox
- **设备：** 桌面（1920x1080）、平板（768x1024）、手机（375x812）

## 测试策略

### 阶段 1：单元测试（核心逻辑）

覆盖 stores 和 services 中的关键逻辑，不依赖浏览器环境。

### 阶段 2：组件测试（UI 组件）

验证组件渲染、事件触发、props 传递。

### 阶段 3：E2E 测试（用户流程）

验证完整的用户操作路径。

### 阶段 4：手动测试（验收）

视觉验证、响应式、跨设备兼容性。

## 通过标准

- 单元测试覆盖率 ≥ 80%
- 所有 P0 测试用例通过
- E2E 核心流程通过
- 无 Critical/Important 级别缺陷
