# 个人导航页 — 测试用例

## 1. 布局系统

### TC-LAYOUT-001：拖拽移动组件
- **前置条件：** 已登录，编辑模式，画布上有 1 个组件
- **操作：** 拖拽组件到新位置
- **预期：** 组件位置更新，布局数据保存
- **优先级：** P0

### TC-LAYOUT-002：调整组件大小
- **前置条件：** 已登录，编辑模式，画布上有 1 个组件
- **操作：** 拖拽组件边缘调整大小
- **预期：** 组件尺寸更新，不小于 minSize
- **优先级：** P0

### TC-LAYOUT-003：响应式断点切换
- **前置条件：** 画布上有组件，已设置 lg 布局
- **操作：** 缩小浏览器窗口至 < 768px
- **预期：** 自动切换到 xs 断点（4 列），组件按 xs 布局渲染
- **优先级：** P0

### TC-LAYOUT-004：缺失断点自动生成
- **前置条件：** 组件只有 lg 布局数据
- **操作：** 切换到 md 断点
- **预期：** 自动从 lg 按比例缩放生成 md 布局，无重叠
- **优先级：** P1

### TC-LAYOUT-005：多端编辑隔离
- **前置条件：** 桌面端已编辑 lg 布局
- **操作：** 在手机端编辑 xs 布局
- **预期：** xs 布局更新，lg 布局不变
- **优先级：** P0

## 2. 组件系统

### TC-WIDGET-001：内置组件渲染
- **前置条件：** 已登录，编辑模式
- **操作：** 从组件库添加搜索框、时钟、天气、书签
- **预期：** 4 个组件正确渲染，功能正常
- **优先级：** P0

### TC-WIDGET-002：搜索框功能
- **前置条件：** 搜索框已添加到画布
- **操作：** 输入关键词，点击搜索
- **预期：** 新标签页打开搜索结果
- **优先级：** P0

### TC-WIDGET-003：搜索引擎切换
- **前置条件：** 搜索框已添加
- **操作：** 点击不同引擎按钮（Google/Bing/Baidu/DuckDuckGo）
- **预期：** 引擎切换，配置持久化
- **优先级：** P1

### TC-WIDGET-004：时钟实时更新
- **前置条件：** 时钟组件已添加
- **操作：** 观察 5 秒
- **预期：** 时间每秒更新，格式正确（HH:MM:SS）
- **优先级：** P0

### TC-WIDGET-005：天气组件配置
- **前置条件：** 天气组件已添加，编辑模式
- **操作：** 配置 API Key 和城市，保存
- **预期：** 显示对应城市天气信息
- **优先级：** P1

### TC-WIDGET-006：书签组件分组
- **前置条件：** 书签组件已添加
- **操作：** 配置分组书签数据
- **预期：** 按分组显示书签，点击可跳转
- **优先级：** P1

### TC-WIDGET-007：第三方组件安装
- **前置条件：** 已登录，编辑模式
- **操作：** 输入 GitHub 仓库地址，预览，安装
- **预期：** manifest 正确解析，组件出现在已安装列表
- **优先级：** P0

### TC-WIDGET-008：第三方组件加载
- **前置条件：** 第三方组件已安装
- **操作：** 将组件拖入画布
- **预期：** 组件从 CDN 动态加载并渲染
- **优先级：** P0

### TC-WIDGET-009：组件配置表单
- **前置条件：** 有 schema 的组件已添加，编辑模式
- **操作：** 点击配置按钮，修改配置，保存
- **预期：** 配置表单自动生成，修改后组件更新
- **优先级：** P1

### TC-WIDGET-010：删除组件
- **前置条件：** 编辑模式，画布上有组件
- **操作：** 点击组件的删除按钮
- **预期：** 组件从画布移除
- **优先级：** P0

## 3. 认证系统

### TC-AUTH-001：登录成功
- **前置条件：** 后端运行，密码已配置
- **操作：** 输入正确密码，点击登录
- **预期：** 登录成功，顶栏显示编辑/退出按钮
- **优先级：** P0

### TC-AUTH-002：登录失败
- **前置条件：** 后端运行
- **操作：** 输入错误密码
- **预期：** 显示"密码错误"提示
- **优先级：** P0

### TC-AUTH-003：登出
- **前置条件：** 已登录
- **操作：** 点击退出登录
- **预期：** 退出成功，顶栏显示登录按钮
- **优先级：** P0

### TC-AUTH-004：无后端模式
- **前置条件：** 后端未启动
- **操作：** 打开页面
- **预期：** 不显示登录按钮，直接显示编辑按钮，所有功能可用
- **优先级：** P0

### TC-AUTH-005：Token 过期
- **前置条件：** 已登录，token 已过期
- **操作：** 执行需要认证的操作
- **预期：** 自动退出登录，显示登录按钮
- **优先级：** P1

### TC-AUTH-006：未登录时只读
- **前置条件：** 有后端，未登录
- **操作：** 查看页面
- **预期：** 组件只读展示，无编辑 UI
- **优先级：** P0

## 4. 数据层

### TC-DATA-001：本地数据持久化
- **前置条件：** 无后端模式
- **操作：** 添加组件，配置，刷新页面
- **预期：** 数据从 IndexedDB 恢复，布局和配置保持
- **优先级：** P0

### TC-DATA-002：后端同步
- **前置条件：** 有后端，已登录
- **操作：** 添加组件，配置
- **预期：** 数据保存到 SQLite，刷新后数据恢复
- **优先级：** P0

### TC-DATA-003：Adapter 自动切换
- **前置条件：** 启动时无后端
- **操作：** 启动后端，登录
- **预期：** 自动切换到 SyncAdapter，重新加载数据
- **优先级：** P1

### TC-DATA-004：跨设备同步
- **前置条件：** 设备 A 已登录并配置
- **操作：** 设备 B 登录同一账号
- **预期：** 设备 B 显示设备 A 的配置
- **优先级：** P1

## 7. 安全测试

### TC-SEC-001：SQL 注入防护
- **前置条件：** 后端运行
- **操作：** 在组件配置中注入 SQL 语句
- **预期：** 数据正常存储，无 SQL 注入
- **优先级：** P0

### TC-SEC-002：JWT 伪造防护
- **前置条件：** 后端运行
- **操作：** 使用伪造的 JWT token 请求 API
- **预期：** 返回 401 未授权
- **优先级：** P0

### TC-SEC-003：CORS 限制
- **前置条件：** 后端运行
- **操作：** 从非授权域名发起请求
- **预期：** 请求被 CORS 策略阻止
- **优先级：** P1

### TC-SEC-004：登录速率限制
- **前置条件：** 后端运行
- **操作：** 连续 6 次输入错误密码
- **预期：** 第 6 次返回 429，提示 15 分钟后重试
- **优先级：** P1

### TC-RESP-003：手机端布局
- **前置条件：** 同上
- **操作：** 在 375x812 分辨率下查看
- **预期：** 4 列网格，组件按 xs 布局排列
